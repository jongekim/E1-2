## 1) 기능 동작 검증

### ✓ 프로그램 실행 시 메뉴가 정상 표시되고, 퀴즈 풀기/추가/목록/점수 확인이 모두 동작하는가?

검증 방법:
```bash
python main.py
```

기대 결과:
- [x] 메뉴 1~5번이 출력된다.
- [x] 1번 퀴즈 풀기 진입 가능
- [x] 2번 퀴즈 추가 진입 가능
- [x] 3번 퀴즈 목록 출력 가능
- [x] 4번 점수 확인 출력 가능
- [x] 종료 후 정상 복귀(5번)

면접 답변 예시:
"프로그램 실행 후 메인 메뉴가 먼저 뜨고, 1~4번 핵심 기능을 각각 독립적으로 실행할 수 있습니다. 기능별 흐름은 QuizGame.run()에서 선택값으로 분기합니다."

### ✓ 정답/오답 판정과 입력 오류 처리(공백/문자/범위 밖/빈 입력)가 정확한가?

검증 방법:
```bash
# 퀴즈 실행 후 아래 입력을 각각 테스트
# 1) 정답 번호 입력
# 2) 오답 번호 입력
# 3) 빈 입력(엔터)
# 4) 문자 입력(abc)
# 5) 범위 밖 숫자(0, 9)
```

기대 결과:
- [x] 정답 입력 시 "정답입니다" 출력
- [x] 오답 입력 시 "오답입니다" 및 정답 번호 안내
- [x] 빈 입력 시 경고 후 재입력 요청
- [x] 문자 입력 시 ValueError 처리 후 재입력 요청
- [x] 범위 밖 입력 시 허용 범위 재안내

면접 답변 예시:
"정답 판정은 Quiz.check_answer()에서 단순 비교로 처리하고, 입력 검증은 utils.py의 get_valid_integer()에서 공백/문자/범위 오류를 모두 잡아서 반복 입력받습니다."

기대 결과:
- [x] 추가한 퀴즈가 재실행 후에도 목록에 남아있다.
- [x] 최고 점수(best_score)가 유지된다.
- [x] 게임 기록(game_history)이 누적된다.
- [x] state.json에 quizzes/best_score/game_history가 저장된다.

면접 답변 예시:
"상태 저장은 save_data(), 복원은 load_data()에서 처리합니다. 퀴즈 추가/결과 반영/종료 시 저장되므로 재실행해도 이전 상태가 유지됩니다."

## 2) 코드 구조 및 설계

### ✓ Quiz, QuizGame, utils, stats를 어떤 기준으로 분리했는가?

확인 파일:
- quiz.py
- quiz_game.py
- utils.py
- stats.py

기대 설명:
- [x] Quiz: 문제 데이터와 동작(출력, 정답검사, 직렬화)
- [x] QuizGame: 메뉴/게임흐름/저장·복원 오케스트레이션
- [x] utils: 입력 검증 공통 함수
- [x] stats: 통계 계산 전담

면접 답변 예시:
"변경 이유가 다른 책임끼리 분리했습니다. 문제 모델은 Quiz, 실행 흐름은 QuizGame, 입력 검증은 utils, 수치 계산은 stats로 분리해 수정 영향 범위를 줄였습니다."

---

### ✓ state.json 읽기/쓰기 흐름을 순서대로 설명할 수 있는가?

검증 포인트:
```text
시작: __init__() -> load_data()
진행: add_quiz()/display_result() 등 -> save_data()
종료: 메뉴 종료/예외 처리 -> save_data()
```

기대 결과:
- [x] 시작 시 복원 흐름 설명 가능
- [x] 상태 변경 시 저장 타이밍 설명 가능
- [x] 종료/예외 시 안전 저장 설명 가능

면접 답변 예시:
"프로그램 시작 시 load_data()로 상태를 복원하고, 상태가 바뀔 때마다 save_data()를 호출합니다. 종료 분기와 예외 처리에서도 저장을 호출해 유실을 최소화했습니다."

### ✓ 채점 방식 또는 퀴즈 구조 요구사항이 바뀌면 어떤 파일부터 수정할지 설명할 수 있는가?

수정 우선순위:
1. 채점 정책 변경
   - quiz_game.py: play_quiz(), display_result()
   - stats.py: 평균/향상도 계산식
2. 선택지 개수 변경
   - quiz.py: 출력/데이터 구조
   - quiz_game.py: add_quiz() 입력 루프
   - utils.py: 입력 범위 검증
   - state.json: 기존 데이터 호환 정책

면접 답변 예시:
"영향 범위가 큰 곳부터 순서대로 바꿉니다. 먼저 게임 규칙을 담당하는 quiz_game.py를 수정하고, 통계와 입력 검증, 데이터 호환을 뒤따라 맞춥니다."
